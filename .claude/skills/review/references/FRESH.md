# Fresh review (first pass over a ticket's PR set)

Loaded from SKILL.md when Step 0 prints `MODE: FRESH`. Workspace is
already prepared: `start-review.sh` ran `prepare-review.sh`, producing
`~/git/defra/trade-imports-animals-workspace/workareas/reviews/EUDPA-XXXXX/`
with `ticket.md`, `repos/`, `file-reviews/` placeholders, and
`.review-meta.json` (detected tech + best-practices paths).

## Model

**Orchestrator:** session role `review-orchestrator` (gate from
`start-review.sh`).
**Workers:** before spawning, resolve once:

```bash
~/git/defra/trade-imports-animals-workspace/tools/agent/resolve-model.sh --worker review-worker --host auto --json
```

Pass `task_slug` from the JSON on each Task spawn when supported; omit
if null. Do **not** pass `model` on the single `CONSISTENCY_REVIEWER`
spawn (inherits session).

## Step 2: Review Each File

**MANDATORY:** create a review for EVERY changed file. No exceptions.

Spawn up to **100 in parallel** via the Task tool with
`subagent_type: general-purpose` and `model:` set to the resolved
`task_slug` (see Model section above; omit if null):

```markdown
Follow the instructions in ~/git/defra/trade-imports-animals-workspace/.claude/skills/review/references/FILE_REVIEWER.md.

**Mode:** FRESH
**Ticket:** EUDPA-XXXXX - [Ticket Summary]

**Your assigned file:**
- Repository: [repo-name]
- Path: [file-path]
- PR: [pr-number]
- Commit: [sha]
```

The reviewer writes findings to the per-file JSON via the
`file-review-add-item.sh` / `file-review-set-verdict.sh` helpers — no
markdown, no placeholder path needed in the spawn prompt.

## Step 3: Verify Coverage

```bash
~/git/defra/trade-imports-animals-workspace/tools/review/verify-coverage.sh EUDPA-XXXXX
```

**Do NOT proceed until 100% coverage.**

## Step 4: Consistency Review

Spawn one `general-purpose` Task subagent:

```markdown
Follow the instructions in ~/git/defra/trade-imports-animals-workspace/.claude/skills/review/references/CONSISTENCY_REVIEWER.md.

**Ticket:** EUDPA-XXXXX - [Ticket Summary]
**Review workspace:** ~/git/defra/trade-imports-animals-workspace/workareas/reviews/EUDPA-XXXXX/

Read .review-meta.json for all repos and PR numbers.
Write _consistency-check.md for every repo listed.
```

Wait for completion. Verify `_consistency-check.md` exists for every repo.

## Step 5: Create Repository Summaries

For each repository, write
`~/git/defra/trade-imports-animals-workspace/workareas/reviews/EUDPA-XXXXX/review.{repo}.md`.
Two sections are deterministic (aggregator output); the rest is your
synthesis from the per-file JSONs and the diff.

```bash
# Write items.{repo}.json from per-file .review.json — call ONCE per repo
# at this step (globally renumbers IDs; preserves no prior disposition).
~/git/defra/trade-imports-animals-workspace/tools/review/aggregate-file-reviews.sh EUDPA-XXXXX --repo {repo} --write-items

# File Analysis Summary table (markdown)
~/git/defra/trade-imports-animals-workspace/tools/review/aggregate-file-reviews.sh EUDPA-XXXXX --repo {repo} --section file-summary

# ## Items markdown view (rendered from items.{repo}.json)
~/git/defra/trade-imports-animals-workspace/tools/review/render-items.sh EUDPA-XXXXX --repo {repo}
```

After this step `items.{repo}.json` is canonical; walker / batch
implementor / refresh mutate it via `review-*.sh` and the `## Items`
markdown view is regenerated by `render-items.sh`.

**Skeleton:**

```markdown
# Repository Review: {repo-name}

**PR:** #{pr-number}
**Commit:** {sha}
**Files Changed:** {count}

## Summary
[2-3 sentences about changes in this repository]

<!-- paste `aggregate-file-reviews.sh ... --section file-summary` output -->

## Positive Observations
[What was done well]

## Test Coverage
- Unit tests: [assessment]
- Integration tests: [assessment]

## Risk Assessment
**Overall Risk:** Low / Medium / High
**Rationale:** [One sentence]

<!-- paste `render-items.sh EUDPA-XXXXX --repo {repo}` output.
     Schema and `|` escape rules: assets/items-table.md.
     Disposition / Status / Notes start blank — walker fills them. -->

## Repository Verdict
**Status:** SAFE / NEEDS ATTENTION / RISKY
```

Skip this step if only one repository is involved.

## Step 5.5: Handoff check

Determine whether the reviewed PRs are yours. For each PR in
`.review-meta.json`:

```bash
~/git/defra/trade-imports-animals-workspace/tools/github/pr-author.sh {repo} {pr}
```

```bash
~/git/defra/trade-imports-animals-workspace/tools/github/whoami.sh
```

- **All PRs authored by you** — print
  `PRs authored by you — proceeding to walker.` and continue to Step 6.
- **Any PR not authored by you** — show a per-PR table
  (`| Repository | PR | Author | Handoff? [Y/n] |`) and prompt. On `Y`
  for at least one PR:

  ```bash
  ~/git/defra/trade-imports-animals-workspace/tools/review/share-review.sh EUDPA-XXXXX [--pr N]
  ```

  Omit `--pr` if every non-yours PR is handed off; pass per-PR for a
  subset. On `n` for all, fall through to Step 6. Capture the printed
  handoff branch URL and PR comment URLs for the Completion Output.

## Verdict Guidelines

| Verdict | Criteria |
|---------|----------|
| **PASS** | All AC met, good quality, adequate tests |
| **PASS WITH NOTES** | AC met, minor non-blocking suggestions |
| **CONCERNS** | Issues to address before merge |
| **FAIL** | Critical bugs, security issues, missing functionality |

`review-mark.sh` per item (or just run the walker and rattle through
with F / W / D keystrokes — normally faster). The walker only shows
items with `disposition: null`, so pre-marked items are skipped.
