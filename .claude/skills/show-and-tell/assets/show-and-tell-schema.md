# show-and-tell state JSON shape

Canonical state file:
`~/git/defra/trade-imports-animals-workspace/workareas/show-and-tell/<id>/state.json`.

`<id>` is the run date (`YYYY-MM-DD`) by default. Mutated only via
`tools/show-and-tell/*.sh` helpers (atomic `jq ... > tmp; mv tmp file`).
The slide markdown view is regenerated by `render-slides.sh` whenever
the JSON changes — never hand-edit `slides.md`.

## Schema

```jsonc
{
  "id": "2026-06-22",                 // run id (defaults to today's date)
  "created_at": "2026-06-22T16:00:00Z",
  "window": {
    "from": "2026-06-08",             // start of the two-week window
    "to": "2026-06-22",               // end (run date)
    "jql": "project = EUDPA AND status = Done AND resolutiondate >= -14d ORDER BY resolutiondate DESC"
  },
  "buckets": {                        // code → display label (fixed)
    "UJ": "User Journey / Skeleton",
    "IA": "Integration / Architecture",
    "TD": "Technical / Delivery enablers"
  },
  "items": [
    {
      "id": 1,                        // 1-based ordinal (resolved-date desc)
      "key": "EUDPA-251",             // Jira key
      "summary": "run e2e via npm run test:docker-compose:ci",
      "type": "Task",                 // Jira issue type
      "status": "Done",
      "resolved": "2026-06-20",       // resolutiondate (date only) or null
      "labels": ["ci"],
      "parent": "EUDPA-215",          // parent epic key or null
      "bucket": "TD",                 // null until classified; one of UJ|IA|TD
      "benefit": "Faster, more reliable test runs for the whole team."
                                      // plain-English benefit; only meaningful
                                      // for TD items; null otherwise
    }
  ]
}
```

## Field rules

- `id` (run) — defaults to the run date; `start-show-and-tell.sh --run-id`
  overrides. Drives the workarea path.
- `window.jql` — the exact query used, recorded for reproducibility.
- `buckets` — fixed three-way taxonomy. The codes (`UJ`/`IA`/`TD`) are
  what `set-bucket.sh --bucket` and the walker keystrokes accept; the
  labels are what the slides print.
- `items[].bucket` — `null` (unclassified) or one of `UJ`/`IA`/`TD`.
  Set by `set-bucket.sh`. `counts.sh` aggregates over this; the
  Completed Work Items slide groups by it.
- `items[].benefit` — only the "Improving how the team delivers" slide
  reads it, and only for `bucket == "TD"` items. Set by
  `set-bucket.sh --benefit`. `null` renders as a "not yet written"
  placeholder.
- `start-show-and-tell.sh` is idempotent: re-running re-fetches the
  window but carries `bucket` + `benefit` forward by `key`, so a
  re-run never discards classification already done.
